### Note on full_proof_suite timeout

The `full_proof_suite.rs` integration tests (added in Wave 16 TL6) run the complete 321-proof database multiple times. 3 of 7 tests passed; the remaining 4 (`full_proof_suite_all_pass`, `full_proof_suite_each_category_passes`, `full_proof_suite_has_exhaustive_and_statistical`, `full_proof_suite_parallel_matches_sequential`) were still executing when the cargo wrapper's 600s timeout hit. These are not failures -- they duplicate the same proofs already validated in the llvm2-verify unit test suite (1,219 tests, 468.61s). Running all proofs 3-4 times requires ~1400-1900s, exceeding the default timeout. This is an expected timeout for an integration suite that stress-tests the full pipeline end-to-end.
