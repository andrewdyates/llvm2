## 3. Fixes Applied

### Fix 1: `verification_runner.rs` doctest timeout

**File:** `crates/llvm2-verify/src/verification_runner.rs`
**Issue:** Module-level doctest called `runner.run_all()` which executes all 395 proof obligations. This takes ~425 seconds for the unit tests and caused a 600s cargo timeout for the doctest runner.
**Fix:** Changed ` ```rust ` to ` ```rust,no_run `. The doctest still compiles (verifying API correctness) but does not execute during `cargo test`. The underlying functionality is fully tested by 12 dedicated unit tests in the `verification_runner::tests` module.

### Fix 2: Missing `ProofCategory::RegAlloc` match arm (concurrent fix)

**File:** `crates/llvm2-verify/src/proof_database.rs`
**Issue:** The `name()` method's match expression did not cover the `ProofCategory::RegAlloc` variant, causing `cargo check --tests` to fail with `E0004: non-exhaustive patterns`.
**Fix:** Added `ProofCategory::RegAlloc => "Register Allocation"` arm. (Applied by concurrent session before this audit's edit.)
