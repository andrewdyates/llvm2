## 2. New Code Quality Review

### 2.1 GVN Pass (crates/llvm2-opt/src/gvn.rs) -- 1187 lines

**Quality: GOOD**

- Scoped value table correctly implements push/pop with cleanup tracking for dominator-tree traversal
- The `kill_loads` method properly saves the load table on first store/call in a scope and restores on pop; subsequent kills in the same scope are handled correctly (idempotent save)
- Commutative canonicalization covers all relevant opcodes: AddRR, MulRR, AndRR, OrrRR, EorRR, FaddRR, FmulRR
- Load value numbering is conservative (any store/call kills all loads) -- correct and safe
- Proof annotations are properly merged/dropped on elimination
- 22 unit tests covering: basic VN, commutative, non-commutative, different operands/opcodes, immediate operands, dominator-based elimination, no-domination safety, load value numbering, store/call kill semantics, scope isolation, transitive VN, chain redundancies, proof preservation/merge/conflict, idempotency, empty function

**Minor observations:**
- FImm hashing uses `to_bits()` which handles NaN correctly (all NaN bit patterns are distinct, which is conservative -- two different NaN bit patterns will get different value numbers even though they're semantically equivalent. This is safe but slightly suboptimal.)
- Recursive dom-tree walk could stack overflow on deeply nested CFGs. Not a concern for practical inputs but noted for completeness.

### 2.2 Tail Call Optimization (crates/llvm2-opt/src/tail_call.rs) -- 810 lines

**Quality: GOOD with caveats**

- Guard conditions (lines 183-222) properly reject: stores between call and ret, nested calls, Release operations, any non-move instruction
- Sibling TCO correctly rejects indirect calls (BLR)
- Backward scan for call (lines 140-153) properly allows only MovR and Copy between call and return
- 20 unit tests covering all expected scenarios

**P3 finding -- incomplete argument shuffle:**
- `apply_self_recursive_tco` (line 295) replaces BL with B to entry block but does NOT perform argument shuffling (`_caller_param_count` unused at line 299). The transformation is only correct when arguments are already in the correct register positions. The doc comment at line 293 mentions "argument shuffle" but the implementation defers it. This limits effectiveness for self-recursive tail calls where arguments are recomputed.

**Design note:**
- Sibling TCO replaces BL with B and keeps Symbol operands. The encoding layer must handle `MachOperand::Symbol` on B instructions (not just on BL). This appears to be handled by the existing encoding infrastructure.
