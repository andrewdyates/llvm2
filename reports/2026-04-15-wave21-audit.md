## P1 Findings (Blocks Critical Path)

### P1-1: Stack Overflow in Tests at Default Stack Size

Two tests overflow the default 8MB Rust test thread stack:

1. `llvm2-codegen::compiler::tests::test_compile_ir_function_add`
2. `llvm2-verify::function_verifier::tests::test_all_pseudo_ops_skipped`

Both pass with `RUST_MIN_STACK=16777216` (16MB). The stack overflow crashes the
entire test binary, aborting all remaining tests in that crate. This means a
plain `cargo test` will show failures in both `llvm2-codegen` and `llvm2-verify`
even though the root cause is stack size, not logic errors.

**Impact:** CI and developer `cargo test` runs fail unpredictably. The abort
kills all tests in the binary, potentially masking real failures.

**Root cause:** Likely deep recursion in the compilation pipeline or large
stack-allocated data structures. The `build_add_test_function()` path and
`verify_function()` + `make_func_with_insts()` path both construct complex
nested structures.

**Fix:** Either:
- Add `RUST_MIN_STACK=16777216` to the test environment (e.g., `.cargo/config.toml`)
- Refactor the deep-recursion paths to use heap allocation or iteration
- Use `std::thread::Builder::new().stack_size(16 * 1024 * 1024)` for those tests

**Files:**
- `crates/llvm2-codegen/src/compiler.rs:419` (`test_compile_ir_function_add`)
- `crates/llvm2-verify/src/function_verifier.rs:748` (`test_all_pseudo_ops_skipped`)

### P1-2: Stale Hardcoded Memory Proof Count (41 -> 62)

Wave 20 added QF_ABV array theory proofs, increasing memory proofs from 41 to
62. Three test assertions still hardcode 41:

1. `crates/llvm2-verify/src/proof_database.rs:643` -- `assert_eq!(count, 41, ...)`
2. `crates/llvm2-verify/src/verify.rs:646` -- `assert_eq!(report.total(), 41)`
3. `crates/llvm2-verify/tests/full_proof_suite.rs:230` -- `assert_eq!(..., 41)`

All 62 memory proofs pass verification. The assertions are purely stale
regression guards.

**Fix:** Update all three assertions from 41 to 62 and update comments.

### P1-3: Stale Hardcoded RegAlloc Proof Count (16 -> 31)

The `regalloc_proofs.rs` module itself documents "31 total: 16 Phase 1 + 15
Phase 2" but external assertions still expect 16:

1. `crates/llvm2-verify/tests/full_proof_suite.rs:235-237` -- expects 16
2. `crates/llvm2-verify/tests/full_proof_suite.rs:280-283` -- expects 16

All 31 regalloc proofs pass verification.

**Fix:** Update assertions from 16 to 31 and update comments.
