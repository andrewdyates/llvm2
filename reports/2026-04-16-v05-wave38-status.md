## 2. Wave 38 Achievements

### 2.1 llvm2-lower Test Migration Complete (commit 4d29582)

The single most impactful commit of Wave 38: **564 test compilation errors reduced
to 0** in llvm2-lower. Commit `[U]431: Part of #289` migrated all llvm2-lower
test code from the old tmir stub API to the real tmir crate API. This was the
critical blocker identified in the Wave 37 report.

**Impact:** Unblocked 771 tests in llvm2-lower (728 pass, 7 fail) and enabled
llvm2-verify tests to compile (2,155 tests).

### 2.2 FP Verification Proofs (commit 279c727)

Added FDIV and FCMP floating-point verification proofs to llvm2-verify. This
extends the formal verification coverage from integer arithmetic to floating-point
operations, specifically:
- FDIV (floating-point division) semantic proof
- FCMP (floating-point comparison) with ordered/unordered variants

### 2.3 x86-64 Encoding Expansion (commit 5c3ee9f)

Added 7 new x86-64 instruction encodings. This was a rescued patch from Wave 37
(W37 techlead). Advances the x86-64 backend from pure scaffolding toward
functional status.

### 2.4 Proof Certificate Chain (commit 33ea4d7)

Added proof certificate persistence infrastructure to llvm2-verify. Proof
certificates can now be serialized and stored, enabling verification results to
survive across compilation sessions. Addresses the concern raised by the
"theatrical proofs" audit finding (#273, now closed).

### 2.5 Documentation Cleanup (commits 5be2b6a, dd09bf6)

- Corrected 8 factual errors in the tMIR migration design doc
- Cleaned up stale stub references in production code comments

## 3. Remaining Test Failures

### 3.1 llvm2-lower: 7 failures in compute_graph GPU proofs

All 7 failures are in `compute_graph::tests` and relate to GPU target
legality assertions:

```
test_annotate_with_proofs_upgrades_legality
test_large_workload_includes_gpu
test_matrix_heavy_with_proofs_recommends_accelerator
test_recommendations_include_parallel_reduction
test_subgraph_proofs_propagate_to_nodes
test_target_recommendations_prefer_gpu
test_with_proof_context_unlocks_gpu
```

**Root cause:** Tests expect `ComputeTarget::Gpu` in legal targets when proof
context is provided, but the implementation returns `[CpuSimd, CpuScalar]`.
This is likely a regression from the tMIR migration affecting proof context
propagation to the compute graph analyzer.

**Severity:** P2. GPU compute dispatch is a future feature (not on critical path).

### 3.2 llvm2-codegen: 140 compilation errors across 12 test files

All errors are stale tmir stub imports (`tmir_func`, `tmir_types`, `tmir_instrs`)
in E2E test files. Distribution:
