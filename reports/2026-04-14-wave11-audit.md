## 2. Test Compilation Fix

**BUG FOUND AND FIXED:** `cargo test --workspace --lib` failed to compile `llvm2-codegen` tests.

Three test functions in `crates/llvm2-codegen/src/pipeline.rs` constructed `PipelineConfig`
without the `verify_dispatch` field added by TL5 in commit `afe0e35`. The field was added to the
struct and its `Default` impl, but three manual struct literals in tests were not updated:

- `pipeline_config_custom` (line 1519)
- `pipeline_config_clone` (line 1529)
- `pipeline_new_with_config` (line 1544)

**Root cause:** TL5 added `verify_dispatch: DispatchVerifyMode` to `PipelineConfig` and updated
the `Default` impl, but the pre-existing tests from TL1 (commit `b2be493`) used struct literal
construction rather than `..Default::default()`. Since TL5's work was in a separate worktree,
the conflict was not caught at merge time (the files did not conflict textually).

**Fix applied:** Added `verify_dispatch: DispatchVerifyMode::FallbackOnFailure` to all three
test struct literals.

## 7. Issues Found

### P1: Test compilation broken on main (FIXED)

`cargo test --workspace` failed to compile `llvm2-codegen` test code due to missing
`verify_dispatch` field in 3 struct literals. This was a merge integration gap between
TL1 (added tests) and TL5 (added field) in Wave 10. **Fixed in this audit commit.**

### P3: 9 dead-code warnings in unified_synthesis.rs

Functions added by TL2 in Wave 10 are not called from any code path. Should be wired
into the public API or prefixed with `_`.

### P3: 4 doc-test `ignore` fences

Doc examples that cannot compile. Should be converted to `no_run` or fixed.

### P3: 5 slow verification tests (>60s each)

The llvm2-verify crate dominates test runtime at ~140s. Consider splitting exhaustive
verification tests into a separate `#[cfg(feature = "slow-tests")]` gate or optimizing
the underlying SMT evaluation.
