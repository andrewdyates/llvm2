## Defects Found and Fixed

### P1: Test compilation broken by cross-agent struct desynchronization (27 errors)

**Root cause:** Wave 9 agents TL2 and TL4 independently added new fields to shared structs but did not update each other's test code:
- **TL4** added `proofs: Vec<TmirProof>` field to `InstrNode` (tmir-instrs) and `Function` (tmir-func)
- **TL2** added `dominant_op: String` to `ComputeNode` and `profitability: Option<ProfitabilityAnalyzer>` to `ComputeGraph` (compute_graph.rs)

The production code compiled cleanly because these agents updated the production constructors, but **test code in other modules** still used the old struct literal syntax without the new fields.

**Files fixed:**
- `crates/llvm2-lower/src/compute_graph.rs` -- 10 `InstrNode` literals + 3 `TmirFunction` literals in test helper functions (`build_large_data_parallel_module`, `build_large_matrix_heavy_module`, `test_target_legality_filters_bitwise_from_ane`)
- `crates/llvm2-lower/src/dispatch.rs` -- 14 `ComputeNode` literals + 4 `ComputeGraph` literals across tests V2, V9, V11, V13, V14

**Fix:** Added missing fields with default values (`proofs: vec![]`, `dominant_op: String::new()`, `profitability: None`) to all 27 test struct literals.

## Recommendations

1. **P2: Add Default impls for test structs.** The `ComputeNode`, `ComputeGraph`, `InstrNode`, and `Function` structs should derive or implement `Default` (or provide builder functions) to prevent this class of cross-agent desynchronization. Every time a field is added, all test literals across the workspace break.

2. **P3: Fix 3 warnings in llvm2-verify.** Unused `load_width` variables in `memory_proofs.rs` and dead `read_array_elem_f64` function in `ane_semantics.rs`.

3. **P3: Add unit tests to tmir stubs.** `tmir-types`, `tmir-instrs`, and `tmir-func` have zero unit tests. While these are stubs, basic serialization round-trip tests would catch field addition issues earlier.
